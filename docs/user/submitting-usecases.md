## Before You Start

### Gather Information

Have this information ready before starting:

- **Business Problem**: What problem does this AI solve?
- **Solution Overview**: How would AI address it?
- **Business Value**: What benefits would result?
- **Team**: Who would be involved?
- **Timeline**: When would you want to start?
- **Estimated Budget**: Rough cost estimate (optional)
- **Stakeholders**: Who needs to approve?

### Template Tips

Use the **Use Case Template** for guidance:
- Available in Submit section
- Shows expected format
- Provides examples
- Can be downloaded

## Step-by-Step Submission

### Step 1: Navigate to Submit

1. Click **"Submit"** in the sidebar
2. Click **"New Use Case"** button
3. You'll be presented with a form

### Step 2: Basic Information

Fill in the basics first:

**Title** (Required)
- Clear, descriptive name for the use case
- Example: "Automated Invoice Processing with AI"
- Limit: 100 characters

**Category** (Required)
- Select primary AI category:
  - Process Automation
  - Predictive Analytics
  - Computer Vision
  - Natural Language Processing
  - Recommender Systems
  - Anomaly Detection
  - Other

**Department** (Required)
- Select your primary department
- More departments can be added later
- Determines initial routing for approval

### Step 3: Business Case

**Problem Statement** (Required)
- Describe the current challenge
- Explain business impact
- Why is this important?
- Word limit: 500 words
- [View example](./examples/problem-statement.md)

**Opportunity** (Required)
- How would AI solve this?
- What would change?
- Word limit: 500 words

**Success Metrics** (Required)
- How will you measure success?
- Provide 2-3 specific metrics
- Should be measurable and time-bound
- Examples:
  - Reduce processing time by 50%
  - Increase accuracy to 99%
  - Cost savings of $500K annually

### Step 4: Resource Planning

**Required Resources**

|Resource | Details |
|---------|---------|
| Data | Volume, types, availability |
| Computing | Infrastructure needed |
| Team | Data scientists, engineers, etc. |
| Budget | Estimated cost |
| Timeline | Months to complete |

**Business Owner** (Required)
- Select from list
- Or add new business owner
- Must have valid corporate email

**Technical Lead** (Optional)
- AI/Data professional
- Can be added later

### Step 5: Stakeholders

**Add Team Members**

1. Click **"Add Stakeholder"**
2. Search by name/email
3. Select their role:
   - Sponsor (executive support)
   - Business Owner (day-to-day)
   - Technical Lead (implementation)
   - Stakeholder (affected by project)
   - Reviewer (provides input)
4. Click **"Add"**

**Multiple Stakeholders**

Add as many as needed:
- Different roles can be assigned
- All will be notified of progress
- Can be updated later

### Step 6: Additional Details

**Project Timeline** (Optional)
- Proposed start date
- Target completion date
- Key milestones

**Budget** (Optional)
- Estimated implementation cost
- Annual operational cost
- Potential ROI

**Dependencies** (Optional)
- Other systems needed
- Data requirements
- Skills/resources needed

**Risk Assessment** (Optional)
- Potential challenges
- Mitigation strategies
- Confidence level

### Step 7: Attachments

**Add Supporting Documents**

1. Click **"Add Attachment"**
2. Select file (max 10MB)
3. Supported types:
   - PDF documents
   - Word documents
   - Presentations
   - Spreadsheets
   - Images
4. Add description
5. Click **"Upload"**

**Useful Attachments**:
- Business case presentations
- Data samples
- Technical specifications
- Feasibility studies
- Market research

### Step 8: Review & Submit

**Review Your Submission**

1. Scroll through the entire form
2. Check all required fields are complete
3. Verify information accuracy
4. Review attached documents

**Submit**

1. Click **"Submit Use Case"** button
2. Confirm submission
3. You'll receive confirmation email
4. Form will be sent to reviewers

## Form Fields Explained

### Required vs Optional

**Required Fields** (must complete):
- Title
- Category
- Department
- Problem Statement
- Opportunity
- Success Metrics
- Business Owner

**Optional Fields** (can complete later):
- Timeline
- Budget
- Attachments
- Risk Assessment
- Additional stakeholders

### Validation Rules

Field validation ensures quality submissions:

| Field | Rule |
|-------|------|
| Title | 5-100 characters, unique |
| Email | Valid corporate email format |
| Metrics | At least 2 measurable metrics |
| Budget | Numeric value if provided |
| Date | Valid date format, future date |

### Character Limits

- Title: 100 characters
- Problem Statement: 500 words
- Opportunity: 500 words
- Descriptions: 1000 words
- Comments: 5000 characters

## Best Practices

### 1. Be Specific

❌ Bad: "Use AI to improve operations"
✅ Good: "Deploy ML model to predict equipment failures 2 weeks in advance, reducing unplanned downtime by 40%"

### 2. Focus on Business Value

❌ Bad: "We want to use machine learning"
✅ Good: "Implement AI-powered customer churn prediction to retain high-value customers, increasing lifetime value by 25%"

### 3. Provide Evidence

- Include data showing the problem
- Cite similar successful projects
- Reference market studies
- Include metrics from pilots

### 4. Be Realistic

- Honest timeline estimates
- Real budget requirements
- Genuine resource needs
- Clear risk acknowledgment

### 5. Get Stakeholder Buy-In

- Involve key stakeholders
- Address their concerns
- Get their input in submission
- Ensure alignment

### 6. Use Clear Language

- Avoid excessive jargon
- Define technical terms
- Use concrete examples
- Keep language professional

### 7. Provide Context

- Explain business background
- Describe organizational impact
- Link to strategic goals
- Show competitive advantage

## After Submission

### Confirmation

You'll receive:
- Confirmation message on screen
- Email to your inbox
- Use case ID for tracking
- Next steps

### Tracking Your Submission

1. Go to **"My Submissions"** section
2. View submission status:
   - Draft (saved, not submitted)
   - Submitted (under review)
   - Approved (ready to progress)
   - Rejected (see feedback)
   - In Progress (became a project)

### Review Process

1. **Initial Review** (1-2 weeks)
   - Admin reviews for completeness
   - Format and quality check

2. **Stakeholder Review** (1-2 weeks)
   - Business leaders review
   - Feasibility assessment

3. **Technical Review** (1-2 weeks)
   - Data/AI team reviews
   - Resource requirements assessed

4. **Final Decision** (1 week)
   - Executive review
   - Approval or feedback

### Feedback

If changes are requested:
1. You'll receive email with feedback
2. Access feedback in your submission
3. Make revisions to the form
4. Resubmit for review
5. Process repeats until approved

### Moving Forward

Once approved:
1. Use case becomes a project
2. Project team is assembled
3. Implementation begins
4. Progress tracked in dashboard

## Troubleshooting

### Form Won't Submit

**Issue**: "Submit" button is grayed out

**Solutions**:
1. Complete all required fields (marked with *)
2. Check field validation messages
3. Ensure proper email format
4. Verify file attachments are valid

### Lost My Draft

**Issue**: My incomplete submission disappeared

**Solutions**:
1. Drafts are saved automatically
2. Check **"My Submissions"** → **"Drafts"**
3. Use browser history to recover
4. Contact support if still missing

### Can't Find Department

**Issue**: My department isn't in the list

**Solutions**:
1. Check exact department name with HR
2. Search with partial name
3. Select closest match
4. Contact admin to add department

### Attachment Won't Upload

**Issue**: File upload fails

**Solutions**:
1. Check file size (max 10MB)
2. Ensure supported file type
3. Try different file format
4. Check internet connection
5. Try different browser

### Didn't Receive Confirmation Email

**Issue**: No email after submission

**Solutions**:
1. Check spam/junk folder
2. Wait up to 5 minutes
3. Check profile email is correct
4. Resend confirmation from website
